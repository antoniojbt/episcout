# Architecture decision records

This folder contains architecture decision records for the specification-first EDA work in `episcout`.

## ADR list

| ADR | Title | Status |
|---|---|---|
| 0001 | Build on `episcout` rather than rewriting | Proposed |
| 0002 | Use a specification-first workflow | Proposed |
| 0003 | Use R, but not pure base R | Proposed |
| 0004 | Support synthetic data before access | Proposed |
| 0005 | Use external real-data fixtures for TDD | Proposed |

## ADR format

Each ADR should include:

- status;
- context;
- decision;
- consequences;
- implementation notes.

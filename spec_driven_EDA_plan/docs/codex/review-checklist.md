# Review checklist

Use this checklist for every PR.

## General

- [ ] PR is small and focused.
- [ ] PR description states motivation and scope.
- [ ] No unrelated refactoring.
- [ ] Existing public behaviour is preserved unless explicitly changed.
- [ ] New behaviour is documented.
- [ ] Errors are clear and actionable.

## R package standards

- [ ] New exported functions have roxygen2 documentation.
- [ ] `NAMESPACE` changes are generated by roxygen2.
- [ ] Tests are added or updated.
- [ ] Examples are simple and deterministic.
- [ ] Dependencies are declared in `DESCRIPTION`.
- [ ] Heavy or optional dependencies are in `Suggests` unless justified.

## Specification-first EDA

- [ ] Function accepts explicit `data` and/or `spec` inputs.
- [ ] Function validates required inputs.
- [ ] Function returns plain R objects.
- [ ] Output is machine-readable.
- [ ] Synthetic-data outputs are clearly labelled when applicable.
- [ ] Real-data and synthetic-data workflows use the same interface where possible.

## Tests

- [ ] Normal case.
- [ ] Empty input where relevant.
- [ ] Missing required columns where relevant.
- [ ] All-missing variables where relevant.
- [ ] Unsupported variable type where relevant.
- [ ] Deterministic synthetic output with fixed seed.
- [ ] Backward compatibility where existing functions changed.

## Performance

- [ ] Avoid unnecessary copies for large objects.
- [ ] Avoid rowwise loops over large datasets unless unavoidable.
- [ ] Plotting can use sampled data.
- [ ] Summaries can run on full data.
- [ ] Expensive work is not repeated in report rendering.

## Documentation

- [ ] `START_HERE.md` reflects the completed PR and next active PR.
- [ ] SDD remains accurate.
- [ ] ADR is added or updated if architectural decision changed.
- [ ] Roadmap is updated if scope changed.
- [ ] User-facing docs distinguish synthetic from real data.

## Commands

Run where practical:

```bash
Rscript -e "devtools::document()"
Rscript -e "devtools::test(reporter = 'summary')"
Rscript -e "devtools::check(manual = FALSE)"
```

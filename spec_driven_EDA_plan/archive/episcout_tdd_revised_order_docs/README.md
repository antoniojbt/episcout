# episcout TDD-first update pack

This pack updates the earlier `episcout` SDD/Codex documentation to make the new spec-first EDA layer test-driven against external, verifiable real-data fixtures.

## What changed

The original plan added the new EDA API first and then tests. This update changes the order:

1. Documentation and baseline remain first.
2. External real-data fixtures come before new spec-first code.
3. Failing fixture-backed tests come before each implementation PR.
4. Expected outputs must be independently computed, not generated by the code under test.
